Proje 10: Uçuşa Hazırız

3D Baskı ve Joystick Modülü ile Simülatör Kontrolcüsü Geliştirme

🎯 Proje Hakkında

Bu proje ile endüstriyel tasarıma ilk adımı atıyoruz! Analog bir joystick modülünden gelen X ve Y ekseni verilerini (ve üzerindeki tıklanabilir buton verisini) aynı anda paketleyerek Unity'ye aktaracağız. Bu verilerle Unity'de bir uçağı uçuracak veya bir robot kolunu yöneteceğiz. Devre elemanlarını gizlemek ve tutuş ergonomisini artırmak için dış kasayı 3D yazıcıdan üreteceğiz.

💡 Hap Bilgi: Joystick modülü aslında içerisinde iki adet potansiyometre (X ve Y ekseni için) ve bir adet push buton (Z ekseni / tıklama) barındıran mekanik bir bileşendir. Arduino bu iki eksenden 0 ile 1023 arasında iki farklı analog değer okur.

🛠️ Gerekli Donanımlar ve Yazılımlar

Donanım Listesi

  • 1x Arduino UNO / Nano
  • 1x İki Eksenli Joystick Modülü
  • Jumper Kablolar (Montaj için dişi-erkek önerilir)
  • Opsiyonel: 3D Basılmış Kumanda Kasası, M3 Vidalar

Yazılım ve Kütüphaneler

  • Unity: System.IO.Ports ve eksen haritalama (InverseLerp)
  • 3D Dilimleyici: Simülatör kasası için 3MF/STL formatındaki dosyaları Bambu Studio gibi bir dilimleyici program ile P1S veya Ender 3 v2 gibi yazıcılara uygun G-Code'a çevirebilirsiniz.

🔌 Bağlantı Şeması

Joystick modülünün 5 pini vardır. Hem analog hem dijital pinleri aynı anda kullanacağız:

Joystick Pini Arduino Pini Açıklama
VCC / 5V 5V Güç girişi
GND GND Topraklama
VRX (veya X) A0 (Analog 0) X Eksenindeki (Sağ-Sol) yatay hareket sensörü
VRY (veya Y) A1 (Analog 1) Y Eksenindeki (İleri-Geri) dikey hareket sensörü
SW (Switch) D2 (Digital 2) Üstüne basıldığında çalışan tıklama butonu

⚠️ Dikkat Edilmesi Gerekenler: Joystick modülünün SW pini için herhangi bir harici direnç kullanmayacağız. Önceki projelerimizde öğrendiğimiz gibi Arduino kodunda pinMode(2, INPUT_PULLUP); kullanarak dahili direnci açacağız.

⚙️ Nasıl Yapılır? (Adım Adım)

Adım 1: Arduino Kodunu Yükleme

Aşağıdaki kod 3 farklı veriyi okur ve "X,Y,Buton" formatında virgülle ayırarak tek bir veri paketi halinde Unity'ye yollar.

const int pinX = A0;
const int pinY = A1;
const int pinButon = 2;

void setup() {
  Serial.begin(9600);
  pinMode(pinButon, INPUT_PULLUP); // Dahili Pull-Up direnci devrede
}

void loop() {
  int degerX = analogRead(pinX);
  int degerY = analogRead(pinY);
  int durumButon = digitalRead(pinButon); // Basılı değilse 1, basılıysa 0

  // Verileri paketle (Örn: 512,512,1)
  Serial.print(degerX);
  Serial.print(",");
  Serial.print(degerY);
  Serial.print(",");
  Serial.println(durumButon);

  delay(20); // 50 Hz veri gönderme hızı (Yumuşak hareket için ideal)
}

Adım 2: Unity Tarafında Veriyi İşleme

Unity'de sahneye bir 3D Uçak veya basit bir Küp ekleyin. Bu nesneye aşağıdaki kodu atayın.

using UnityEngine;
using System.IO.Ports;

public class SimulatorUcus : MonoBehaviour {
    SerialPort port = new SerialPort("COM3", 9600); 
    public float donusHizi = 45f;
    public float hareketHizi = 10f;

    void Start() {
        if(!port.IsOpen) port.Open();
        port.ReadTimeout = 50;
    }

    void Update() {
        if (port.IsOpen) {
            try {
                string gelenVeri = port.ReadLine();
                string[] parcalar = gelenVeri.Split(',');

                if (parcalar.Length == 3) {
                    // Verileri dönüştür
                    int joyX = int.Parse(parcalar[0]);
                    int joyY = int.Parse(parcalar[1]);
                    int butonDurumu = int.Parse(parcalar[2]);

                    // 0-1023 arası değerleri Unity'nin standart giriş eksenlerine çevir (-1.0 ile 1.0 arası)
                    float eksenX = Mathf.InverseLerp(0, 1023, joyX) * 2 - 1;
                    float eksenY = Mathf.InverseLerp(0, 1023, joyY) * 2 - 1;

                    // Küçük donanımsal dalgalanmaları yoksaymak için ölü bölge (Deadzone) ekliyoruz
                    if(Mathf.Abs(eksenX) < 0.1f) eksenX = 0;
                    if(Mathf.Abs(eksenY) < 0.1f) eksenY = 0;

                    // Uzay Gemisini/Uçağı Kontrol Et
                    // Y eksenini uçağın burnunu kaldırmak (Pitch), X eksenini yana dönmek (Roll) için kullanıyoruz
                    transform.Rotate(eksenY * donusHizi * Time.deltaTime, 0, -eksenX * donusHizi * Time.deltaTime);
                    
                    // Nesneyi sürekli ileri doğru uçur (Motor İtici Gücü)
                    transform.Translate(Vector3.forward * hareketHizi * Time.deltaTime);

                    // Joystick butonuna basıldığında (Ateş etme mekanizması)
                    if (butonDurumu == 0) {
                        Debug.Log("Lazer Ateşlendi!");
                        // Instantiate(lazerMermisi, atisNoktasi.position, transform.rotation);
                    }
                }
            } 
            catch (System.Exception) { /* Gecikmeleri yoksay */ }
        }
    }

    void OnApplicationQuit() {
        if(port.IsOpen) port.Close();
    }
}

🚀 Tavsiyeler ve Basit Öneriler

✨ Geliştirme Önerisi 1: Donanım Mimarisi: Simülatör kasanızı 3D yazıcıda basarken içerisine birden fazla joystick (Örneğin sol ele motor gücü, sağ ele yön kontrolü için) ve ekstra silah anahtarları (havacılık tipi Toggle Switch) ekleyerek profesyonel bir HOTAS (Hands On Throttle-And-Stick) sistemi üretebilirsiniz.
✨ Geliştirme Önerisi 2: Kalibrasyon (Deadzone): Ucuz joystick modülleri tam ortadayken (boşta) 512 yerine 500 veya 530 gibi sapmalı değerler üretebilir. Bu durum Unity'deki uçağınızın siz dokunmasanız bile hafifçe dönmesine (Drift) sebep olur. Unity kodunda yer alan if(Mathf.Abs(eksenX) < 0.1f) eksenX = 0; satırı, bu küçük sapmaları filtreleyen "Ölü Bölge" filtresidir. İhtiyaca göre bu 0.1f değerini artırıp azaltabilirsiniz.
